# SC-TH 主线说明

当前产品主线已经从 LaTeX review workspace 重构为“本科论文 Word 正文审查稿导出器”。

## 当前主路径

1. 首页只保留 `SC-TH` 和一条主输入框；输入框内保留 `+` 上传入口、右侧弱化提交箭头，下方保留错误提示与清空入口。
2. 用户上传 `.docx` 或粘贴文本后，先触发结构与合规预检。
3. 系统返回预检确认弹窗，展示结构预览、阻塞项、警告项和信息项。
4. 只有阻塞项清零后，才允许确认导出。
5. 导出阶段用浅蓝海浪进度反馈，完成后自动下载 `.docx` 正文审查稿。

## 本轮固定口径

- 不是学校官方工具
- 不自动生成学校统一正式封面
- 正式统一封面需另行套用
- 当前导出顺序采用 PDF 规范口径：中文摘要 -> 英文摘要（页标题 `Abstract`） -> 目录 -> 正文与注释 -> 参考文献 -> 附录 -> 致谢
- 目录为必备项，输出为 Word 可更新目录字段

## 规则层

- 阻塞项：不满足时禁止导出
- 警告项：允许继续，但提示风险
- 信息项：解释当前识别结果、封面口径和导出预期

当前阻塞重点是：题目、中文摘要、英文摘要、正文结构、正文长度、参考文献和模板可用性。

## 当前边界

- 仅支持本科论文入口
- 仅公开 `.docx` 主产物
- 不保留旧 review workspace 作为主路径
- 不承诺图片、表格、脚注和复杂排版高保真迁移
- 不承诺引用链与参考文献条目 100% 自动对齐
- 不把注释、图题、表题伪装成“已严格自动支持”

## 验收方式

当前主线的验收已经分成三层：

1. 产品流转验收：首页 -> 预检 -> 导出
2. 自动化合规检查：`scripts/check_docx_compliance.py`
3. 样例审查：`examples/compliance/` + `docs/compliance/scnu-undergraduate-export-audit-report-v1.md`
