本表以以下材料为最高依据：

1. [《华南师范大学本科生毕业论文（设计）手册》](./sources/华南师范大学本科生毕业论文（设计）手册.doc)

如两份材料存在冲突，本轮默认以 PDF 规范为主口径执行。

## 冲突点与本轮采用口径

| 项目 | 手册 | PDF 规范 | 当前采用口径 |
| --- | --- | --- | --- |
| 目录 | 视篇幅和结构决定是否需要 | 必备项 | 必备项 |
| 文档顺序 | 封面、中文摘要、英文摘要、目录、正文、注释、附录、参考文献、致谢 | 封面、中文摘要、外文摘要、目录、正文与注释、参考文献、附录、致谢 | 不生成正式学校封面；正文审查稿顺序采用 `中文摘要 -> 英文摘要（页标题 Abstract） -> 目录 -> 正文与注释 -> 参考文献 -> 附录 -> 致谢` |

## 状态定义

- `已支持`：已有稳定实现，并进入自动化检查
- `部分支持`：有实现，但仍依赖人工复核或只做到风险提示
- `不支持`：当前不自动保证，必须显式披露

## 当前结论

当前项目已经不再是“只有结构像论文的导出器”，而是具备明确合规边界的正文审查稿导出工具：

- 硬性页面与样式项：`已支持`
- 注释、图表题注、复杂富文本迁移、参考文献细项：`部分支持` 或 `不支持`
- 对不能自动保证的项目，当前已通过预检提示、合规脚本结果和文档说明同步披露
