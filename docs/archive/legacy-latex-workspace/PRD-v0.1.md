# PRD v0.1 - scnu-thesis-portal

## 1. 项目定位

`scnu-thesis-portal` 当前定位为：面向华南师范大学论文写作场景的模板整合与使用入口，后续可演进为一个降低论文格式处理成本的工具型 Portal。

这一阶段的核心不是做复杂功能，而是先把模板、说明、产品结构和后续方向沉淀清楚。

## 2. 核心问题

当前用户在论文模板使用上普遍面临以下问题：

- 模板来源分散，缺少统一入口。
- 使用门槛高，尤其对第一次接触 LaTeX 的学生不友好。
- 不同模板之间的差异不透明，用户不知道该选哪套。
- 缺少明确的使用建议、场景说明和上手路径。
- 如果后续要做网站或工具，当前缺少稳定的产品结构和需求沉淀。

## 3. 目标用户

### 3.1 普通学生使用者

主要需求是快速判断该用哪套模板、如何开始、有哪些常见风险。

### 3.2 熟悉 LaTeX 的进阶使用者

主要需求是快速比较模板差异、评估可维护性，并选择适合自己的基础模板继续定制。

### 3.3 后续维护者 / 开发者

主要需求是获得清晰的资料组织、产品边界、技术方向和迭代路线，避免重复摸索。

## 4. MVP 范围

MVP 先聚焦“可用的论文模板入口仓库 / Portal 雏形”，范围包括：

- 模板整理与保守归档
- 模板差异说明
- 模板选型建议
- 基础导航与文档入口
- 为后续 Web Portal 铺设结构基础

MVP 不以自动化能力为核心，而以降低理解成本和选型成本为核心。

## 5. 明确不做

在 v0.1 / MVP 阶段，以下内容明确不做：

- 不承诺自动格式修复
- 不承诺全文论文生成
- 不做复杂在线编辑器
- 不做与学校官方系统的深度整合
- 不假设当前模板已经过官方规范最终验证

## 6. 信息架构初稿

第一版信息架构建议如下：

- 首页 / Portal 入口
- 模板对比
- 使用指南
- 常见问题
- 示例与下载
- Roadmap

在仓库阶段，上述结构先以 README 和 docs 体现；后续再转化为静态站点页面。

## 7. 技术方向初稿

从实现成本、可维护性和阶段目标看，当前技术方向应保持克制：

- 当前仓库先以内容整理和结构化文档为主。
- 后续若做 Portal，前端优先考虑静态站点方案。
- 暂不引入重后端，不在早期做复杂服务架构。
- 优先选择低成本、可快速部署、可长期维护的实现路径。

## 8. 版本路线

### v0.1

资料整理、仓库初始化、盘点文档、PRD 初稿。

### v0.2

完善 README / Docs，并补出模板对比页与更清晰的使用建议。

### v0.3

实现最小 Portal 页面雏形，提供基础导航与模板入口。

### v1.0

让真实用户能够快速完成模板选择并开始上手使用。

## 9. 当前成功标准

如果 v0.1 达成，应该能满足以下条件：

- 新仓库结构清晰，原始材料不混乱。
- 用户第一次进入仓库时，能在 README 中快速理解项目现状。
- 维护者能通过 `docs/inventory.md` 和本 PRD 理解当前判断与下一步方向。
- 仓库已完成独立初始化并可继续迭代。
